Introduction

What is CS|Studio Map Guide Tool?

The Map Guide Tool is a browser-based editor for CS2 KV3 annotation files — the format used by the game's workshop annotation system to display in-game guide overlays. It lets you load raw annotation files, automatically apply formatting rules, manually edit any node, reorder your lineup list, and export a clean, game-ready file.

Everything runs in the browser. No account required, no data is sent anywhere, and files never leave your computer.

Getting Started

Quick Start

1

Choose your guide mode

Use the ⊟ Single Guide / ⊞ Dual Guide toggle in the top-right to choose between a single annotation file or split CT + T files. Most creators start with Dual Guide for separate CT-side and T-side guides.

2

Load your file(s)

Drag a .txt KV3 annotation file onto the drop zone, or click the zone to browse. In Dual Guide mode, load a CT-side file on the left and a T-side file on the right.

3

Process

Click ⚡ Process Both in the header (or ⚡ Process on an individual panel) to automatically apply all formatting rules — colors, throw types, tag stripping, and text node merging.

4

Review and edit

Click any node row to expand it and edit its title, description, color, grenade type, and more. Drag the ⠿ handle to reorder nodes.

5

Export

Click ↓ Export on the panel to download your processed file. Drop it into your CS2 workshop annotation folder.

Getting Started

Loading Files

Each panel accepts one .txt KV3 annotation file. There are two ways to load:

🖱️

Drag and drop

Drag your annotation .txt file directly from File Explorer onto the panel's drop zone. The zone highlights in blue when a file is hovering over it.

📁

Click to browse

Click anywhere on the drop zone to open a file picker. Select your .txt annotation file.

✕

Clear a file

Once a file is loaded, a ✕ Clear button appears in the panel header. Click it to unload the file and return the panel to its empty state so you can load a different file.

ℹ️
The Map Guide Tool expects standard CS2 KV3 annotation files as exported by the annotation_create console commands. The file should begin with a <!-- kv3 encoding:... --> header and contain MapAnnotationNode blocks.
Core Features

Guide Modes

Use the mode toggle in the top-right corner to switch between how many guides you're working with.

⊞ Dual Guide

Two panels side by side — CT-side on the left, T-side on the right. Load a separate annotation file for each team. Nodes can be moved between panels. Process and export each file independently.

⊟ Single Guide

One full-width panel labeled "Guide." Use this when creating a unified annotation file that isn't split by team. The "Move to CT/T" option is hidden in this mode.

💡
Switching modes doesn't discard loaded files — your data is preserved. You can toggle back and forth freely.

Filtering and Search

The toolbar above the list provides filters to isolate specific node types — All, Grenade, Position, Spot, or Text. Use the search box to filter by title or description text in real time.

⚠️
Drag-to-reorder works across all views, but moving a node while a filter is active will renumber based on the full list order — not just what's visible. The index badge always reflects the true file position.
Core Features

Auto-Processing

Clicking ⚡ Process runs a five-pass formatter over all nodes. Each pass handles a different node type in order.

Pass 1 — Grenade main nodes

Assigns the correct color based on grenade type (smoke, flash, molotov, etc.). Extracts the throw type from the last parenthetical in the title — for example, B Site Smoke (6.5s) (Jump Throw) becomes title B Site Smoke (6.5s) with description Jump Throw. Sets the JumpThrow flag automatically when the throw type implies a jump.

Pass 2 — Grenade aim targets

Inherits color from the parent grenade node. Fills an empty aim instruction description with a hint from a nearby text node, or falls back to the throw type.

Pass 3 — Position nodes

Assigns color by keyword in the title: sound / cue / listen → purple, wallbang / bang → gray, plant / planting → coral, everything else → gold. Strips group tag prefixes like (LSW) or [CF]. Merges the description of any text node within 150 units into an empty position description, then removes that text node.

Pass 4 — Spot nodes

Sets all spot nodes to gray regardless of their original color.

Pass 5 — Text node cleanup

Text nodes that are not titled Tip or Guide By are merged into the nearest aim target or position node (within 150 units) and then removed. Standalone text nodes and tip nodes are kept as-is.

💡
Processing is non-destructive — it works on a copy of the node data. You can process multiple times without corrupting your file, and you can always manually override anything it changed.
Core Features

Editing Nodes

Click any node row to expand its editor panel. All fields are editable at any time — before or after processing.

FieldDescription
Title The main text displayed in-game for this node.
Description Secondary text shown below the title. For grenade mains, this is the throw type instruction. For aim targets, it's the aim hint.
Color Preset swatches for every standard node color, plus a custom color picker. Click a swatch to select it, or use the picker for any hex value.
Grenade Type grenade main only Changing this also updates the color to match the standard scheme for that grenade type.
Jump Throw grenade main only Toggles the JumpThrow flag. Enable this for any lineup that requires a jump while throwing.
Enabled Controls whether the node is visible in-game. Uncheck to hide a node without deleting it.

Actions

Save

Save

Commits your field changes to the node and refreshes the list. Remember to Save before clicking away.

→ Move

Move to CT / Move to T

Dual Guide mode only. Moves this node to the other panel, inserting it before the Guide By attribution node. The other panel must have a file loaded.

Delete

Delete

Permanently removes the node from the list after a confirmation prompt. This cannot be undone — reload the original file if you need to recover a deleted node.

Core Features

Reordering Nodes

The order of nodes in the file determines which ones appear in online/competitive mode. Nodes 0–29 (the first 30) are visible everywhere. Nodes 30 and above only appear in Practice.

To reorder, grab the ⠿ handle on the left side of any node row and drag it up or down to its new position. The list updates immediately and all index numbers renumber automatically.

💡
Use the orange divider as a visual guide. Drag your highest-priority lineups — the ones you want available in competitive warmup or online play — above the divider into slots 0–29.
ℹ️
Grenade nodes typically come in groups of three: a G main node, a GA aim target, and a GD destination. Try to keep these groups together when reordering so the lineup displays correctly in-game.
Core Features

Copying Coordinates

Every node that has a position recorded shows clipboard copy buttons on its row. These let you instantly teleport to a lineup position in CS2 for review or recording.

How to use in CS2

1

Open a practice server

Launch CS2, open the developer console (` key), and load the map with map de_dust2 (or your map).

2

Enable cheats

Run sv_cheats 1 in the console — required for teleport commands.

3

Copy and paste

Click the tp button on any node in the tool, switch to CS2, open the console, paste with Ctrl+V, and press Enter. You'll teleport directly to that lineup position facing the aim spot.

// Example — copied from a smoke node's tp button:
setpos -1245.31 892.40 64.03; setang -14.22 67.50 0.00
⚠️
Clipboard access requires a secure (HTTPS) context. If you're running the tool locally via file://, copy buttons will silently fail. Host the tool on GitHub Pages or another HTTPS server for clipboard to work.
Core Features

Exporting Files

Click the ↓ Export button on any panel to download the processed annotation file. The output includes the full KV3 wrapper (MapName, WorkshopSubmissionID, ScreenText) so it can be loaded directly into CS2 without any manual editing.

The exported filename appends _processed before the .txt extension so it doesn't overwrite your source file.

Where to put the file

Place the exported .txt in your CS2 annotation folder:

// Default path:
Steam\steamapps\common\Counter-Strike Global Offensive\game\csgo\annotations\

Then load it in-game with the console command:

map_annotations_load filename_without_extension
ℹ️
To publish to the Steam Workshop, create a new workshop item of type Annotation Guide, upload your exported file, and paste the resulting WorkshopSubmissionID back into the Settings field before re-exporting.
Reference

Node Types

Each node row displays a short type badge so you can identify it at a glance without expanding it.

BadgeTypeDescription
G Grenade — Main The throw position for a grenade lineup. Contains position, angles, grenade type, throw type, and JumpThrow flag.
GA Grenade — Aim Target The crosshair aim point for the lineup. References a master grenade node via MasterNodeId.
GD Grenade — Destination Where the grenade lands. Used as a visual reference in the guide overlay.
P Position A standing position — pre-aim spot, sound cue position, wallbang standing spot, or plant position.
S Spot — Main A spot marker used in wallbang groups or similar. Always gray.
SA Spot — Aim Target The aim point for a wallbang. References a master spot node.
T Text A floating text label. Kept if titled Tip or Guide By; otherwise merged into nearby nodes during processing.
L Line A connector line between nodes. Rarely used directly.

Settings

Open Settings via the ⚙ Settings button in the header. All settings are saved automatically to your browser's local storage and persist across sessions.

Colors tab

Change the color assigned to each node type. Click a swatch to open a color picker. Changes take effect on the next processing run, or immediately for newly created nodes.

Typography tab

Control the font sizes and fade distances written to exported files. These values map directly to KV3 fields:

SettingKV3 FieldEffect
Title size / Desc sizeFontSizeText scale in the in-game overlay (default: 125 / 75)
Title in / Desc inFadeInDistDistance at which text begins to appear
Title out / Desc outFadeOutDistDistance at which text fades out. -1 = always visible

Throw Types tab

Maps raw throw type text found in node titles to the formatted label written to the Description field. The left column is what the tool looks for (case-insensitive); the right column is what gets written. Add entries for any custom throw type abbreviations used in your recordings.

Tag Prefixes tab

Group tags like (LSW), [CF], or (MDSFL) are used during recording to link related nodes together. The tool strips these from titles during processing. Add any custom tags you use here so they are cleaned automatically.

ℹ️
Use the Reset Defaults button at the bottom of the Settings modal to restore all values to factory defaults. This cannot be undone.

When recording a grenade node, stand at the throw position and face the aim spot before running annotation_create grenade. The node captures both your position and your view angle, which is exactly what the tool copies to clipboard.
